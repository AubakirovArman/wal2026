# Docs Layout

This directory now has a modular layout for future work.

Rules:
- Keep new markdown files at 200 lines or fewer when practical.
- One experiment or one milestone note per file.
- Keep one index file that aggregates links for each area.
- Treat the current top-level files as stable entry points until their content is migrated.

Folders:
- `overview/` — high-level design notes and concept splits.
- `roadmap/` — milestone planning and execution checklists.
- `decisions/` — ADR-style decisions, ideally one file per decision.
- `diary/` — chronological experiment journals.
- `experiments/` — one file per experiment result, grouped by area.

Top-level entry points kept for compatibility:
- `concept.md`
- `plan.md`
- `decisions.md`
- `dev_diary_ru.md`
