# Experiment Notes

This folder is the modular home for experiment write-ups.

Structure:
- `runtime/` — kernels, microbench, same-encoding runtime gates
- `quality/` — PPL, eval, accuracy, parity checks
- `wal/` — grammar, token, and WAL-level analysis
- `templates/` — reusable note templates

Rules:
- one experiment per file
- include command, inputs, outputs, and conclusion
- link the experiment back into `plan.md` or `dev_diary_ru.md`
