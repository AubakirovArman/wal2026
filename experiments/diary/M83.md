# M83: Phase 11 — Ecosystem Integration

## Date
2026-04-20

## Goal
Integrate WAL with the broader ML ecosystem: HF Hub, ONNX export, model merging.

## What was built

### HF Hub Integration (`src/wal/v1/hub.py`)
- `WALModelCard` — metadata card (base model, encoder config, metrics)
- `extract_wal_state_dict()` — serialize complete WAL model to dict
- `load_wal_state_dict()` — deserialize and reconstruct
- `push_wal_model()` — upload to HF Hub (WAL blobs + safetensors + config)
- `pull_wal_model()` — download and reconstruct from HF Hub

### ONNX Export (`src/wal/v1/onnx_export.py`)
- `export_wal_simple()` — pre-decode WAL weights → standard ONNX Linear
- `export_wal_native()` — WAL decode as ONNX ops (Gather + Mul + Reshape + MatMul)
- `verify_onnx_export()` — bit-exact verification against PyTorch output

### WAL-aware Mergekit (`src/wal/v1/mergekit.py`)
- `MergeConfig` — configuration for merge strategies
- `merge_wal_models()` — program-level merge with 4 strategies:
  - **soup**: atom/coeff-level merge (fastest, no re-encode)
  - **linear**: decode → average → re-encode
  - **slerp**: spherical linear interpolation (2 models)
  - **ties**: trim, elect sign, merge
- `merge_task_vectors()` — task arithmetic on WAL models
