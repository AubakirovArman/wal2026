# M84: KV-cache WAL Probe

## Date
2026-04-23

## Goal
Understand KV-cache structure from Llama 70B and evaluate whether WAL encoding is suitable for KV-cache compression.

## Method
1. Load Llama 3.3 70B on GPUs 2,3
2. Run forward pass with 8-token prompt
3. Extract KV-cache from all 80 layers
4. Analyze statistics: distribution, correlation, entropy, sparsity
5. Compare KV-cache to weight distribution

## Key Results

### Structure
- 80 layers, 8 KV heads per layer, head_dim=128
- KV-cache shape per layer: [1, 8, seq_len, 128]
- Total KV elements for 8 tokens: 1,310,720 (655K keys + 655K values)

### Distribution
| Metric | Keys | Values | Weights (q_proj) |
|--------|------|--------|------------------|
| min | -18.88 | -8.13 | -0.21 |
| max | 19.13 | 7.78 | 0.24 |
| mean | -0.017 | 0.0001 | 0.0000 |
| std | 1.80 | 0.20 | 0.014 |

**Critical finding**: Keys are much more spread out (std=1.8) than values (std=0.2). Values are highly concentrated near zero.

### Temporal Correlation (Layer 40, Head 0)
- K adjacent-pos correlation: **0.6047** — strong
- V adjacent-pos correlation: **0.3366** — moderate
- K dim self-correlation: 0.3696
- V dim self-correlation: 0.3625

**Finding**: KV-cache has real temporal structure. Adjacent positions are correlated. This is different from weights which are i.i.d.

### Entropy
| Metric | Keys | Values | Weights |
|--------|------|--------|---------|
| Unique values | 4,443 | 4,958 | — |
| % unique | 0.68% | 0.76% | — |
| Entropy (256 bins) | **5.43 bits** | **3.47 bits** | 5.05 bits |

**Critical finding**: V-cache entropy is 3.47 bits — significantly LOWER than weights (5.05 bits). This means V-cache can potentially be compressed with FEWER bits than the 12-bit floor that applies to weights.

### Sparsity
- K |x| < 0.01: 3.58%
- V |x| < 0.01: 18.00%
- W |x| < 0.01: 56.50%

Values are sparser than keys but less sparse than weights.

### Hypothesis Evaluation
| Hypothesis | Keys | Values |
|------------|------|--------|
| H1: Smoother than weights | NO (std 1.8 > 0.014) | NO (std 0.2 > 0.014) |
| H2: Temporal correlation | YES (0.60) | YES (0.34) |
| H3: Lower entropy than weights | NO (5.43 > 5.05) | **YES (3.47 < 5.05)** |

### Size Analysis (8 tokens)
- At bf16: 2.5 MB
- At 12-bit WAL: 1.9 MB
- Potential savings: 25%

For long context (32K tokens): ~10 GB at bf16 → ~7.5 GB at WAL (savings of 2.5 GB).

## Key Insights

1. **V-cache is the target**: Lower entropy (3.47 bits) and moderate temporal correlation make V-cache a better compression target than K-cache.

2. **Temporal correlation is exploitable**: Adjacent positions are correlated. This suggests delta-encoding or prediction-based approaches could work.

3. **Per-head structure varies significantly**: Some heads have std=1.45, others std=1.83. Universal atom tables may not work — per-head or per-layer tables may be needed.

4. **K-cache is harder**: High entropy (5.43 bits) and high variance (std=1.8). May need the full 12-bit budget.

5. **Scale matters**: For 8 tokens, KV-cache is tiny (2.5 MB). Compression only matters for long context (4K+ tokens).
