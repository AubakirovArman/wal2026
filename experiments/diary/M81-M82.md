# M81-M82: Phase 10 — Meta-Learning

## Date
2026-04-20

## Goal
Fine-tune WAL programs instead of weights. Enable model soups at program level and task-specific adapters.

## What was built

### Core Components (`src/wal/v1/meta.py`)

1. **WALProgramAdapter** — LoRA-style residual adapter
   - `lora_A [out, rank] @ lora_B [rank, in]` scaled by α/rank
   - Only adapter parameters are trainable; base WAL weight frozen
   - Merge method for inference-time weight fusion

2. **WALCoeffAdapter** — Learned coefficient offsets
   - `adapted_coeff = base_coeff + learned_delta`
   - Operates in coefficient space (more WAL-native)
   - 0.4% of full weight parameters for C=8

3. **WALAtomAdapter** — Selective atom perturbations
   - Adapts subset of atoms (default: first 8)
   - 2.0% of full weight parameters for 8 atoms

4. **program_soup()** — Model soup at program level
   - Mean: average atom_ids and coeff_ids
   - Majority: majority vote per position
   - Weighted: weighted average

5. **evolve_programs()** — Genetic algorithm
   - Population of program buffers
   - Fitness = negative MSE
   - Operators: mutation (flip atom/coeff), crossover (single-point)
   - Elitism: keep top-k individuals

### Integration (`src/wal/v1/nn.py`)
- `WALCachedLinear.set_adapter()` — attach/detach adapter
- Adapter applied during decode, cached with weight
- Gradient flow verified through adapter parameters

## Key insight
Meta-learning on WAL programs is fundamentally different from weight fine-tuning:
- **Weight fine-tuning**: Changes all N×M parameters → expressive but large
- **Program fine-tuning**: Changes program selection (atom_ids, coeff_ids) or small adapters → compact
- **Coefficient adaptation**: Most WAL-native; changes multipliers, not basis vectors
- **Genetic evolution**: Discrete optimization; useful when gradients don't apply
