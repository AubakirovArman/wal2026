## Critical Insights

### 1. WAL = Editability, Not Compression
```
Format     Size       Ratio    Purpose
bf16       16.06 GB   1.0x     Baseline
int8        8.03 GB   2.0x     Quantization
int4        4.02 GB   4.0x     Aggressive quantization
WAL        11.26 GB   1.4x     STRUCTURAL EDITABILITY ← NEW
```
WAL adds 43% size overhead but enables **deterministic, reversible parameter surgery**.

### 2. Global Atoms = Production Ready
- **PPL impact: +0.03** (+0.3%) — statistically neutral
- **Storage: 262 KB** atom table vs 59 MB (per-layer)
- **Speed: 216s** encode vs 304s (per-layer)
- **225 layers served by 1 atom table**

### 3. Edit Workflow Proven
```
WAL model → decode() → dense nn.Linear → LoRA edit → merge → re-encode → WAL model
```
- 10/10 contrafactuals survive round-trip
- PPL penalty: +2.90 (acceptable for surgical edits)
- **Must use manual training loop** (Trainer breaks WAL layers)

### 4. Fundamental Limits Discovered
| Limit | Evidence | Implication |
|-------|----------|-------------|
| Discrete program space non-differentiable | M117: soup = 6.4e13 PPL | Merge in weight space only |
| Greedy encode near-optimal | M122: GA 170M% worse | No need for program search |
| Atoms not semantic | M121: entropy 0.966, top-3 = 3% | Atoms = basis, not concepts |
| Re-encode lossy for fine edits | M119: 0%→50% retention | Knowledge "snaps back" on re-encode |
| Small data overfits | M120: PPL 43→246 | Behavioral patches need scale |
