# 14 — Glossary

## A

**Atom** — A learned scalar value (L0) or composite expression (L1+). The primitive unit of WAL programs.

**Atom table** — The collection of all atoms available to a model or layer. In WAL v2, a flat list of K scalars. In WAL v1, a hierarchical structure with L0 base atoms and L1 composites.

**AtomLibrary** — A persistent collection of pre-trained atom tables that can be saved, loaded, and transferred across models.

## B

**Backend** — A hardware-specific implementation of WAL decode. Backends exist for CPU (NumPy), CUDA (PyTorch GPU), MPS (Metal), ROCm (AMD), and WebGPU.

**Binary format** — The compact, machine-readable serialization of WAL programs. See `format.py`.

**Bits per weight (bpw)** — The storage cost per weight. WAL achieves ~12 bpw (8 bits atom_id + 4 bits coeff_id).

## C

**Coefficient** — A learned scale factor that multiplies an atom. In WAL v2, C=16 discrete levels.

**Codebook** — The set of unique programs that actually occur in an encoded tensor. Typically ~1,300 unique programs per 67M weights.

**Compression ratio** — The ratio of dense model size to WAL-encoded size. WAL achieves 1.33× vs bf16.

## D

**Decode** — The process of converting WAL programs back to dense weights. `weight = atom[atom_id] * coeff[coeff_id] + residual`.

**Debugger** — The interactive tool for stepping through WAL programs, setting breakpoints, and inspecting execution.

**DRL v2** — Dynamic Route Language version 2, the failed predecessor to WAL. Used ternary routes, ladders, and stages.

## E

**Encode** — The process of converting dense weights to WAL programs. Two-pass: k-means for atoms, Lloyd-Max for coefficients.

## F

**Flat atoms** — Precomputed scalar values for all atoms including L1 composites. Used for fast decode.

## H

**Hierarchical atoms** — The WAL v1 feature where atoms can be composites of other atoms (L0 base + L1 composites).

**Hot route** — A frequently occurring route in DRL v2. Attempted caching optimization that failed due to flat distribution.

## I

**ISA** — Instruction Set Architecture. The formal definition of WAL operations and data types.

## K

**K** — The number of base atoms. Typically K=256 (8 bits).

## L

**Ladder** — The per-row calibrated decision tree used in DRL v2. Abandoned in WAL.

**L0 atom** — A base scalar atom.

**L1 atom** — A composite atom built from weighted combinations of L0 atoms.

**Lloyd-Max** — The iterative quantization algorithm used to build coefficient tables and assign weights to atoms.

## M

**Meta-learning** — Fine-tuning WAL programs or adapters instead of raw weights.

## P

**Palette** — A learned codebook of route segments in DRL v2. Abandoned in WAL.

**Perplexity (PPL)** — The primary quality metric. Lower is better. Only full-model PPL is valid.

**Program** — The discrete representation of a single weight: `(atom_id, coeff_id, residual)`.

**ProgramBuffer** — The data structure storing all programs for a tensor.

## R

**RelMSE** — Relative mean squared error. **Permanently discarded** as unreliable. Only full-model PPL matters.

**Residual** — The correction term added when `atom * coeff` is not accurate enough.

**Route** — A ternary sequence through a ladder in DRL v2. Abandoned in WAL.

## S

**Scalar ISA** — The WAL instruction set where atoms are scalars (not vectors). The only proven foundation.

**Stage** — A group of weights sharing a codebook in DRL v2. Abandoned in WAL.

## T

**Task vector** — The difference between a fine-tuned model and its base model. Used in model merging.

**Ternary** — The `{-1, 0, +1}` coefficient set used in DRL v2. Too restrictive for vectors.

**Tile** — A 16×16 block of weights in DRL v2. Abandoned in WAL.

## V

**Vector atom** — An atom that is a high-dimensional vector rather than a scalar. Attempted in WAL-1 and failed catastrophically.

**VRE** — Vector Route Encoder. A block-quantization approach that produced PPL >7000 despite perfect single-layer metrics. The most important failure in the project.

## W

**WAL** — Weight Assembly Language. A language for neural network weights where every weight is a program.

**WALLinear** — A PyTorch `nn.Module` that stores weights in WAL format and decodes on-the-fly.

**WALParameter** — A parameter-like object that stores weights in WAL-encoded format with lazy decode.

**WAL v1** — The hierarchical atom version. Adds semantic structure on top of v2.

**WAL v2** — The production codec. Flat scalar atoms, single-call programs.

**WAL-0** — The first WAL prototype with multi-call programs (PUSH_ATOM, MUL, ADD, STOP).

**WGSL** — WebGPU Shading Language. The shader format generated by the WebGPU backend.
